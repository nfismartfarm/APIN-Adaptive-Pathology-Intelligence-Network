### Defect-2 (HIGH): Subagent 8.5 directory-hash algorithm under-specified

- **Class:** ambiguity (algorithm imprecision can produce false drift)
- **Location:** master prompt section 8 subagent 8.5 (sacred-guardian)
- **Quoted defect:** *"Directories: build a JSON object `{relative_path: sha256_of_file}` for every file under the directory (sorted by relative_path for determinism), then `sha256` the JSON serialization."*
- **Why this is a defect:** "JSON serialization" does not specify separator format. Default `json.dumps` uses `(", ", ": ")`; compact JSON uses `(",", ":")`. Different separators produce different bytes and therefore different SHA256 values. Two implementations following the verbal description in good faith will disagree, producing FALSE DRIFT alerts at every sacred-guardian run.
- **Severity:** HIGH
- **Suggested fix:** specify the exact algorithm in the master prompt: `json.dumps(file_hashes, sort_keys=True, separators=(",", ":")).encode("utf-8")` and `relative_path = relpath.replace(os.sep, "/")`. Mirror this in `.claude/sacred_manifest.json` metadata.
- **Resolution status:** Phase 0 added `directory_hash_algorithm_canonical` field to the manifest documenting the exact algorithm. Master prompt subagent 8.5 still has the verbal version; queued for `T-EARLY-MP`.
