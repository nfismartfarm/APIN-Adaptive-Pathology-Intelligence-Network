## Defect-19 (HIGH) — Planner output format unspecified: prose cards vs checkbox template

**Location:** master prompt Section 4.3 task 5 + Section 8.2 planner body

**Problem:** Section 4.3 task 5 mandates "checkbox format" with `- [ ] Task ID: T-001` example. Section 8.2 planner body lists required fields but says nothing about format. Planner produced prose cards (`### T-IMPL-1a` headings) instead. Phase 4 step 9 ("Update tomato_plan.md checkboxes") is mechanically unfollowable without checkboxes.

**Suggested fix:** Add to Section 8.2: *"Output format is the checkbox template from master prompt Section 4.3 task 5. Each task card begins with `- [ ] Task ID: T-NNN`. Do not use `###` heading cards."*

## Defect-20 (HIGH) — No requirement for 9-column summary table in planner output

**Location:** Section 4.3 task 5; Section 8.2 planner body

**Problem:** User Phase 2 entry instruction required 9-column task summary table. Master prompt mentions no summary table at all. Planner produced 5-column table, flagged as FAIL B1.

**Suggested fix:** Add to Section 4.3 task 5: *"Append a 9-column summary table at the end of `tomato_plan.md`: Task ID | Title | Owner subagent | Prerequisites | Spec sections | Files to create | Acceptance criteria | Estimated complexity | Priority."*

## Defect-21 (MEDIUM) — Phase 3 entry gate missing `spec_changelog.md` condition

**Location:** Section 4.3 Phase 3 entry text

**Problem:** Section 4.3 Phase 3 says only "Entry: Phase 2 complete and user approved" — no gate checklist. DEC-012 added 3 entry conditions (T-IMPL-5a, BLK-004 spec_changelog, Defect-16 patch). Master prompt has no place to record these. Plan's `T-PHASE-3-PRECONDITIONS` lists 3 gates but omits spec_changelog. Auditor B3 PARTIAL FAIL.

**Suggested fix:** Replace Phase 3 entry with explicit 5-item checklist: (1) Phase 2 user-approved, (2) T-IMPL-5a complete, (3) T-IMPL-5b complete, (4) Master-prompt Fix-16 applied, (5) `spec_changelog.md` contains all required BLK entries.

## Defect-22 (MEDIUM) — Effort ceiling 1-3h not enforced; planner body permits 4h

**Location:** Section 4.3 task 4 vs Section 8.2 planner body field definition

**Problem:** Section 4.3 task 4 says "1-3 hours each." Section 8.2 lists effort field as "(1h, 2h, 4h)" — explicitly permitting 4h. Planner produced 5 tasks at 4h.

**Suggested fix:** Change Section 8.2 to "(1h, 2h, 3h — maximum 3h; split if larger)."

## Defect-23 (MEDIUM) — `spec_changelog.md` format does not handle interpretation-only entries

**Location:** Section 11.4b spec_changelog format template

**Problem:** Section 11.4b template requires "Original text" and "New text" fields, implying text substitution. BLK-004's resolution (DEC-012 option A) does NOT change spec text — only establishes that line 4117 is authoritative over line 5558 typo. Format does not accommodate "authoritative interpretation without text change." Result: spec_changelog.md still has zero entries despite DEC-012 requiring one.

**Suggested fix:** Add SPEC-INT-NNN (interpretation) variant to Section 11.4b with fields: Conflicting locations / Authoritative location / Non-authoritative location / BLK reference / Defect ID / Author subagent.

## Defect-24 (MEDIUM) — Severity ordering convention for fix lists not in master prompt

**Location:** Section 4.3 Phase 2 / Section 8.2 planner body

**Problem:** User instruction "HIGH-then-MEDIUM-then-LOW" not in master prompt. Planner placed Fix-16 (HIGH, blocks Phase 3) at position 16 of 17 after MEDIUM items. Auditor B2 FAIL.

**Suggested fix:** Add to Section 8.2: *"For fix-list tasks (T-EARLY-MP and similar), order items HIGH first, then MEDIUM, then LOW. Within HIGH, items labeled 'BLOCKS Phase N' precede other HIGH items."*

## Defect-25 (LOW) — DEC format does not show multi-BLK resolution structure

**Location:** Section 11.4 DEC format

**Problem:** DEC-012 resolved 4 BLKs in one entry. Format template has single `Spec section:` field. DEC-012 had to improvise nested bullets. No `Resolves BLKs:` field exists for cross-referencing.

**Suggested fix:** Add multi-BLK note to Section 11.4: *"For multi-BLK DECs, repeat `Spec section:` per section and add `Resolves BLKs: BLK-NNN, BLK-MMM, ...` field."*

## Defect-26 (LOW) — Phase 2 exit criteria omit artifact-completeness gates

**Location:** Section 4.3 Phase 2 exit + phase-exit-auditor agent definition

**Problem:** Section 4.3 Phase 2 exit: "plan is comprehensive; every spec contract has at least one task." No mention of T-PHASE-3-PRECONDITIONS checklist completeness or BLK-resolved-via-DEC-012 verification. Auditor's check #9 was improvised from DEC-012 context, not from master-prompt text — non-reproducible.

**Suggested fix:** Extend Phase 2 exit to: *"(1) plan is comprehensive — every spec contract has ≥1 task; (2) T-PHASE-3-PRECONDITIONS contains all DEC-N-required gate conditions; (3) all BLKs surfaced during planning are logged."*
