## Defect-27 (HIGH) — No procedure for incomplete exit gate

Master prompt Section 4 says "STOP and report" at each phase exit but never specifies which subagents must run, what "complete" means, or what to do if only some auditors fire. Round 1 and Round 2 both fired only phase-exit-auditor (1 of 5 expected). Documented disclosure does not retroactively satisfy a missing instruction.

**Fix:** Add to Section 4: *"At every phase exit, fire these five subagents as a parallel batch before consolidating: phase-exit-auditor, PVA, PDA, anti-cheat-inspector, sacred-guardian. Wait for all five to return before consolidating. If any fails, write to tomato_blockers.md and do not advance."*

## Defect-28 (HIGH) — No authority/procedure for plan-edit patches

Master prompt section 4.2 says planner produces the plan but never says what happens when the plan has errors at the exit gate. Round 1/2/3 inline patches were applied by main thread without master-prompt authorization. PVA SD-6-new flagged this as silent deviation.

**Fix:** Add to Section 4.2: *"If exit gate finds errors in tomato_plan.md: mechanical errors (wrong threshold, inverted annotation) may be patched inline by main thread after logging in tomato_log.md and tomato_decisions.md. Structural errors require re-invoking planner. Exit gate must re-fire after any patch."*

## Defect-29 (MEDIUM) — No authorized escalation for spec-body verification

When auditor finds inter-artifact discrepancy (summary vs plan), master prompt has no procedure. BLK-009 Defect-9.2 required ad-hoc user authorization for spec-body re-read.

**Fix:** Add to Section 18: *"Spec-citation verification: if an auditor finds two project artifacts disagree on a spec claim, main thread may read relevant spec body section directly to resolve. Quote verbatim in log entry. If spec body itself is ambiguous, file BLK."*

## Defect-30 (MEDIUM) — phase-exit-auditor severity taxonomy unguided

Phase-exit-auditor judges findings as BLOCKING vs ADVISORY without explicit prompt guidance. Round 3 "27 vs 25" cosmetic judgment was reasonable but auditor-discretion-dependent.

**Fix:** Add to phase-exit-auditor agent definition: *"Findings are BLOCKING (must fix before phase advance) or ADVISORY (gate may proceed). BLOCKING: any phase-exit checklist criterion unfulfilled; HIGH-severity finding from any agent; sacred drift. ADVISORY: cosmetic typos with no behavioral impact; closed log gaps."*

## Defect-31 (MEDIUM) — Section 8 missing 3 agents

Section 8 lists 8 agents (8.1-8.8). Project uses 11. The 3 missing: phase-exit-auditor, prompt-validator (PVA), prompt-defect-analyst (PDA). Phase 0 task 7 says "Create the 8 subagents" — fresh-session would miss the 3.

**Fix:** Add Sections 8.9, 8.10, 8.11 with full agent definitions. Update Phase 0 task 7 to "Create the 11 subagents."

## Defect-32 (MEDIUM) — Spot-check missing spec-citation accuracy step

Phase 1 spot-check ("verify summary is faithful") missed BLK-009 Defect-9.2 (architectural invariant inverted). Sampled sections 17/23/11 didn't cover function signatures, dataclass fields, or index-space invariants.

**Fix:** Extend Section 4.1 task 2: *"Spot-check must include: (a) one section with function signature/dataclass — verify field names quoted verbatim; (b) one section with cross-section architectural invariant; (c) one section with numeric threshold — verify value and inequality direction verbatim."*

## Defect-33 (LOW) — "STOP for ambiguity" vs "small ambiguity" inconsistent

Section 5 Rule D says "any ambiguity → write blockers and STOP." Section 10 says "small ambiguities → read more carefully." Boundary undefined.

**Fix:** Reconcile: *"Small ambiguity = spec has a single unambiguous statement that resolves; read more text. Blocker-worthy = (a) two sections conflict, (b) reading more doesn't resolve, (c) decision between valid interpretations needed."*
