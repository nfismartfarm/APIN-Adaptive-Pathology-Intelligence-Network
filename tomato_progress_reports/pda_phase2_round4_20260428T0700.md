## Most consequential defects (HIGH)

### Defect-37 (HIGH) — DEC-015 Phase 4 implementer protocol absent from master prompt

DEC-015 documents the Phase 4 implementer methodology (plan is scaffolding for build order/dependencies; spec body is contract for code shape). The master prompt has no instruction telling Phase 4 implementer subagent to follow this protocol. A fresh-session re-create reading only the master prompt would not know.

**Fix:** Add to Section 4 Phase 4: "Task cards in `tomato_plan.md` are authoritative for build order, spec section pointers, file targets, dependencies, acceptance criteria pointers. They are NOT authoritative for verbatim contract details. Before writing code, read the spec body sections listed in the task card."

### Defect-42 (HIGH) — Section 8.4 implementer body contradicts DEC-015

Section 8.4 (implementer agent) currently says: *"You read spec section summaries from `.claude/spec_summaries/` rather than the full spec (to keep context focused)."* DEC-015 requires implementer to read spec body for code-shape decisions. **Direct contradiction.** A fresh-session implementer following Section 8.4 reproduces the 63-66% defect rate.

**Fix:** Update Section 8.4 to: *"For code-shape decisions (function signatures, dataclass fields, threshold values, algorithm steps), read the spec body section directly. Summaries are for context and dependency orientation only."*
