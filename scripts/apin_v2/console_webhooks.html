How to test a webhook
  1. Set up a free public receiver at webhook.site or requestbin.com. Copy the unique URL they give you.
  2. Click + new webhook above, paste that URL, subscribe to one or more events, and save. You will see the signing_secret exactly once — save it.
  3. Click the test button on the new row. APIN fires one synchronous webhook.test POST and shows the HTTP status the receiver returned. A 200 here means the receiver got the event and acknowledged it.
  4. To verify the signature: on each delivery APIN sends an APIN-Signature header shaped t=<timestamp>,v1=<hex>. Compute HMAC-SHA256(secret, "<timestamp>.<raw-body>") and compare. Reject if the timestamp is older than 5 minutes (replay protection).

How verification works

Each delivery has an APIN-Signature header in the form t=<unix-ts>,v1=<hex>. The hex is HMAC-SHA256(secret, "<ts>.<event-id>.<raw-body>"). Compare in constant time. See the quickstart for sample verifier code.

Rotate signing secret

Mints a new signing secret. During the grace window, deliveries are signed with the new secret but receivers can verify against EITHER the new or the old secret (via the v1_prev= portion of APIN-Signature).

0 = old secret stops immediately (drops in-flight deliveries). Default 86400 = 24 h. Max 30 days.
