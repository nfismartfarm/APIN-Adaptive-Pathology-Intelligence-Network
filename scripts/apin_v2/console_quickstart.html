Get started with the APIN API

five minutes from "no key" to "diagnosed leaf", with copy-paste-ready samples in five languages

APIN serves plant-disease inference over a small REST surface. The flagship endpoint, /api/predict/full, accepts a leaf photo and returns a routed diagnosis with confidence. This guide walks you through your first call, the response shape, error handling, retries, and the conformal-set concept that is unusual to APIN.

1Mint your first key

Open the API keys page and click new key. You will be asked to confirm your password (sudo step-up). Pick at minimum the predict:write scope — that is the scope /api/predict/full needs to authorise.

Tokens are shown ONCE. Save your token to a password manager or environment file immediately. APIN never stores plaintext — if you lose the value, your only option is to rotate, which mints a new one.
Your key looks like apin_live_<43 base62 chars> — a prefix that names the environment (live or test) followed by 43 base62 characters of entropy.

The rest of this guide assumes the token is in an environment variable named APIN_KEY. Set it with export APIN_KEY=apin_live_… (Unix) or $env:APIN_KEY = "apin_live_…" (PowerShell).

2Make your first call

Send any leaf photo (JPEG / PNG / WebP, up to ~8 MB) as the image multipart field. Five languages, copy-paste-ready:

terminal
curl -X POST https://dxv-404-apin.hf.space/api/predict/full \
  -H "Authorization: Bearer $APIN_KEY" \
  -F "file=@leaf.jpg"
predict.py
import os, requests

resp = requests.post(
    "https://dxv-404-apin.hf.space/api/predict/full",
    headers={"Authorization": f"Bearer {os.environ['APIN_KEY']}"},
    files={"file": ("leaf.jpg", open("leaf.jpg", "rb"), "image/jpeg")},
    timeout=30,
)
resp.raise_for_status()
print(resp.json())

predict.go
package main

import (
    "bytes"; "fmt"; "io"; "mime/multipart"
    "net/http"; "os"
)

func main() {
    f, _ := os.Open("leaf.jpg")
    defer f.Close()

    body := &bytes.Buffer{}
    mw := multipart.NewWriter(body)
    part, _ := mw.CreateFormFile("file", "leaf.jpg")
    io.Copy(part, f)
    mw.Close()

    req, _ := http.NewRequest("POST",
        "https://dxv-404-apin.hf.space/api/predict/full", body)
    req.Header.Set("Authorization", "Bearer "+os.Getenv("APIN_KEY"))
    req.Header.Set("Content-Type", mw.FormDataContentType())

    resp, err := http.DefaultClient.Do(req)
    if err != nil { panic(err) }
    defer resp.Body.Close()
    out, _ := io.ReadAll(resp.Body)
    fmt.Println(resp.Status, string(out))
}
HTTPS only. Plaintext HTTP is rejected at the edge — if you see a connection error against http://, swap to https://. APIN does not redirect; that would leak the Bearer token to the network.

3Read the response

What the tiers mean

  • 1A: routed + high confidence. Render diagnosis as the primary answer.
  • 2A, 3A, 4A: specialist + ensemble ran but disagree or saw low confidence; surface the conformal set as alternatives.
  • ROUTER_REJECTED: routed crop is outside this API's scope. data.diagnosis is null; data.output_message tells the caller what to do.
  • TOMATO_UNAVAILABLE / TOMATO_INFERENCE_ERROR: tomato pipeline failed; retryable.
Out-of-distribution flag. data.is_ood is computed via Mahalanobis distance to the training feature cloud (data.mahalanobis_distance). When true, treat the diagnosis as a hint, not an answer: the image may not be a leaf at all.

4Handle errors

Account endpoints (keys, sudo, settings, webhooks, alerts) follow the spec §3 envelope — on error, ok is false and error.code tells you what went wrong. The most common codes:

StatusCodeWhat it means & what to do
400invalid_parameterBody or query param failed validation. Read error.message — it names the field. Do not retry without changing the input.
401invalid_or_missing_tokenThe Authorization header is missing or malformed. Verify your key is current and the header is Bearer <key>.
401key_expiredThis key's expires_at has passed. Mint a new one and disable the old.
401key_disabledAn operator disabled this key. Check the Console for context.
403missing_scopeThis endpoint needs a scope this key does not have. error.details.scope names the missing scope. Edit the key on the Console — or mint a new one with the right scope.
403sudo_requiredMutating endpoints on the Console need sudo. Browser flow: the Console handles the modal; programmatic flow: POST /api/account/sudo with your password to refresh.
403origin_not_allowedThis key has an origin_allowlist and your Origin header is not on it. Add the origin in the Console or unset the allowlist.
404not_foundThe resource ID does not exist or is not yours. Re-check the ID.
409already_rotatingA key rotation is already in progress. Wait or revoke the in-flight rotation.
429rate_limitedPer-key or org-wide rate ceiling hit. Honour the Retry-After response header before retrying. See section 5.
429quota_exceededThe daily quota for this key (or org) is exhausted. Resets at midnight UTC. Raise the cap in settings or wait.
503service_unavailableAn upstream model is warming or the service is restarting. Safe to retry after a short backoff.

Full-error-handling sample (Python)

predict_with_errors.py
import os, time, requests
from requests.exceptions import Timeout, ConnectionError

URL = "https://dxv-404-apin.hf.space/api/predict/full"
KEY = os.environ["APIN_KEY"]

def predict(image_path, max_attempts=4):
    for attempt in range(max_attempts):
        try:
            with open(image_path, "rb") as f:
                r = requests.post(URL,
                    headers={"Authorization": f"Bearer {KEY}"},
                    files={"file": ("leaf.jpg", f, "image/jpeg")},
                    timeout=30)
        except (Timeout, ConnectionError) as e:
            # Network blip: short backoff, retry
            time.sleep(2 ** attempt)
            continue

        if r.status_code == 200:
            return r.json()
        if r.status_code == 429:
            # Honour Retry-After if present
            wait = int(r.headers.get("Retry-After", 5))
            time.sleep(wait + min(2 ** attempt, 10))
            continue
        if r.status_code == 503:
            # Model warming or restart: short backoff
            time.sleep(2 ** attempt)
            continue
        # 400 / 401 / 403 / 404 / 409 → not retryable
        body = r.json()
        raise RuntimeError(f"APIN {r.status_code} {body.get('error', {}).get('code')}: {body.get('error', {}).get('message')}")
    raise RuntimeError(f"Gave up after {max_attempts} attempts.")

print(predict("leaf.jpg"))

5Retry & rate limits

APIN sets a Retry-After header on every 429 and most 503s. Always honour it before retrying. Layer your own exponential-backoff on top so you do not stampede when the suggested wait is short.

Headers to read on every response

  • Retry-After — seconds to wait before the next attempt. Present on 429 and most 503.
  • X-RateLimit-Remaining — remaining requests in this window (per-key).
  • X-RateLimit-Reset — Unix timestamp when the per-key bucket resets.
  • X-Quota-Remaining-Today — remaining requests in the daily quota (per-key).

Recommended backoff schedule

  1. 1st failure: wait Retry-After (or 1 s if absent), retry once.
  2. 2nd failure: wait Retry-After + 2 s jitter, retry.
  3. 3rd failure: wait Retry-After + 4 s jitter, retry.
  4. 4th failure: give up, surface the error to the caller.

Total wait is bounded; the call site sees one outcome (success or final error) in < 30 s typical.

Do not retry 4xx errors automatically. 400 / 401 / 403 / 404 / 409 mean "this request is wrong" — retrying will only repeat the same failure and burn your rate budget. Only 429, 503, and transport-level errors (timeouts, connection resets) are safely retryable.

6Idempotency for safe POST retries

If a POST request fails ambiguously (network reset, timeout) you do not know whether the server processed it. Resending it could create a duplicate — e.g. a second key, a second webhook. Defend against this by sending an Idempotency-Key header on POSTs that create resources.

create_key_idempotent.py
import uuid, requests

idem = str(uuid.uuid4())   # keep this stable across retries

r = requests.post(
    "https://dxv-404-apin.hf.space/api/account/keys",
    headers={
        "Authorization": f"Bearer {APIN_KEY}",
        "Idempotency-Key": idem,
        "X-Console-Csrf": csrf_token,
    },
    json={"name": "backend-prod-2026-05", "scopes": ["predict:write"]},
    timeout=15,
)
# If this request fails with a network error, re-call with the SAME idem.
# The server returns the original response, not a duplicate creation.

APIN remembers the idempotency key for a sliding 24-hour window; resending the same key returns the cached response with status 208 Already Reported.

7Conformal sets

APIN does not just give you diagnosis: "early_blight". It also gives you a conformal_set: the smallest set of class labels that contains the true label with at least 90% calibrated probability. If the model is sure, the set has one element. If it is unsure, the set grows to two or three.

Use the set length as a confidence shortcut. A single-element conformal set means "show the diagnosis as the answer". A two- or three-element set means "show alternatives, ask the user to verify" — the UI does this with a stacked card.

Conformal sets are calibrated on a held-out test split per crop family, not by the per-image confidence score. That makes them robust to over-confident extrapolation outside the training distribution.

  • If you only need a single prediction, read diagnosis. conformal_set is a power-user feature.
  • If you build a UI on top, render the set when it has more than one element — the user gets useful uncertainty information without seeing raw probability percentages.

8Libraries

APIN does not ship official SDKs — the surface is small enough that requests / fetch is the right level. The examples above use only stdlib clients in each language. If you want a typed client, generate one from the OpenAPI spec:

  • OpenAPI 3.1 schema — download and feed to openapi-generator (Python, Java, Go, Rust, TypeScript…).
  • Browse interactive docs — every endpoint, every field, with try-it forms.
  • Sandbox playground — click-to-test against your live keys, no terminal needed.
